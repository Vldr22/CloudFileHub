Файловый хостинг с S3 хранилищем, JWT-аутентификацией и антивирусным сканированием.

---

**Аутентификация**
* Публичный доступ — просмотр списка файлов и скачивание без регистрации
* Вход через `POST /api/auth/login` — JWT токен устанавливается в HttpOnly cookie
* Все остальные защищённые endpoints требуют валидный JWT токен в cookie

---

**Уровни доступа**
* `ROLE_USER` — загрузка 1 файла, удаление только своих файлов
* `ROLE_ADMIN` — загрузка до 5 файлов, удаление любых файлов, управление пользователями, просмотр аудит-логов и статистики
* Публичный доступ — просмотр и скачивание файлов без регистрации

---

**Формат ответов**

Все endpoints возвращают `CommonResponse<T>`:
```json
{
  "data": {},
  "status": "SUCCESS",
  "timestamp": "2026-03-11T03:41:17.639342344"
}
```

Ответ с ошибкой:
```json
{
  "status": "ERROR",
  "problemDetail": {
    "type": "about:blank",
    "title": "Not Found",
    "status": 404,
    "detail": "File not found"
  },
  "timestamp": "2026-03-11T03:41:17.639342344"
}
```

---
**Ошибки**
* `400` — ошибка валидации или неверный тип параметра
* `401` — токен отсутствует или истёк
* `403` — недостаточно прав или лимит загрузки исчерпан
* `404` — файл или пользователь не найден
* `409` — конфликт (username уже занят, файл дублируется)
* `413` — файл превышает максимальный размер (30MB)
* `500` — ошибка S3 или внутренняя ошибка сервера
